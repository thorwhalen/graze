graze.util.clog(condition: bool, *args, log_func: ~typing.Callable = <built-in function print>, **kwargs)[source]

Conditional log

>>> clog(False, "logging this")
>>> clog(True, "logging this")
logging this

One common usage is when there’s a verbose flag that allows the user to specify whether they want to log or not. Instead of having to litter your code with if verbose: statements you can just do this:

>>> verbose = True  # say versbose is True
>>> _clog = clog(verbose)  # makes a clog with a fixed condition
>>> _clog("logging this")
logging this

You can also choose a different log function:

>>> _clog = clog(True, log_func=lambda x: print(f"hello {x}"))
>>> _clog("logging this")
hello logging this

graze.util.human_readable_bytes(num_of_bytes: int, *, approx_marker='~', n_digits=4, base=1000, units=('B', 'KB', 'MB', 'GB', 'TB', 'PB'))[source]

Convert a number of bytes into a human-readable format with units.

Parameters:

  • num_of_bytes (int) – The number of bytes.

  • approx_marker (str, optional) – Marker indicating approximation. Defaults to “~”.

  • n_digits (int, optional) – Number of significant digits in the output. Defaults to 4.

  • base (int, optional) – Conversion base, 1024 or 1000. Defaults to 1000.

  • units (tuple, optional) – Units for conversion. Defaults to (‘B’, ‘KB’, ‘MB’, ‘GB’, ‘TB’, ‘PB’).

Returns:

Human-readable string representing the number of bytes.

Return type:

str

Examples: >>> human_readable_bytes(123456) ‘~123.456KB’ >>> human_readable_bytes(123456789) ‘~123.457MB’

Using base=1000 (default) is the International System of Units (SI) standard. But base=1024 is a common practice in computing.

>>> human_readable_bytes(123456789, base=1024)
'~117.738MB'

>>> human_readable_bytes(1234567890123)
'~1.235TB'

Tip: Use functools.partial to customize the function to your needs.

>>> from functools import partial
>>> readable = partial(
...     human_readable_bytes,
...     approx_marker='approximately ',
...     units=(' bytes', ' kilobytes')
... )
>>> readable(123456)
'approximately 123.456 kilobytes'

Note how the function starts using the scientific notation when it runs out of units.

>>> readable(123456789)
'approximately 1.235e+08 kilobytes'
